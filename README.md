# Timetable Extractor Backend

This is a Modular Monolith FastAPI backend designed to authenticate users via Firebase, handle KYC image uploads, and extract structured timetable data using OCR.

## Current State

We are building this application step-by-step. Currently, **Foundation** and **Module 1 (Auth)** are complete.

### Completed Modules
- **Foundation**: FastAPI app, global exception handler, CORS, environment configurations.
- **Module 1 (Auth)**: Firebase ID token verification, custom JWT issuing, and route protection dependency (`get_current_user`).

---

## Getting Started

### Prerequisites
- Python 3.10+
- A Firebase project with a generated Service Account Private Key.

### 1. Environment Setup
Create a `.env` file in the root directory (use `.env.example` as a blueprint):
```env
APP_NAME=check
DEBUG=True
JWT_SECRET_KEY=your-secure-random-secret
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
FIREBASE_SERVICE_ACCOUNT_PATH=./firebase-adminsdk.json
MAX_UPLOAD_SIZE_MB=10
ALLOWED_ORIGINS=["http://localhost:3000"]
```

Make sure you place your downloaded `firebase-adminsdk.json` in the root folder alongside the `.env` file.

### 2. Install Dependencies
```bash
pip install -r requirements.txt
```

### 3. Run the Server
```bash
uvicorn app.main:app --reload
```
The server will start at `http://localhost:8000`. Interactive API documentation is automatically available at `http://localhost:8000/docs`.

## How to Test the API

You can test the endpoints using **Postman** or directly in your browser at [http://localhost:8000/docs](http://localhost:8000/docs).

### 1. Test the Server (Health Check)
- **Method**: `GET`
- **URL**: `http://localhost:8000/`
- **Expected Output**: 
  ```json
  {
      "status": "healthy",
      "app": "check",
      "version": "0.1.0"
  }
  ```

### 2. Test the Auth Guard (`/auth/me`)
This route is protected and expects a valid JWT. We can test that the security guard is working by intentionally failing it.
- **Method**: `GET`
- **URL**: `http://localhost:8000/auth/me`
- **Headers**: Leave empty.
- **Expected Output**: `403 Forbidden` (This proves the route is protected).
- **Test with Fake Token**: Add a Header `Authorization: Bearer fake-token`.
- **Expected Output**: `401 Unauthorized` (`"Could not validate credentials."`).

### 3. Test Firebase Login (`/auth/firebase-exchange`)
Because this endpoint requires a *real* Firebase ID token generated by the Google/Firebase login popup on a frontend, you cannot easily test it with a fake string in Postman. 
- **Method**: `POST`
- **URL**: `http://localhost:8000/auth/firebase-exchange`
- **Body** (JSON): 
  ```json
  {
      "token": "eyJhbGciOiJSUz..." 
  }
  ```
*Note: To test this end-to-end, we will connect a React/Next.js frontend to trigger the Firebase SDK, grab the token, and send it here.*

### 4. Get Saved Timetable (`/timetable/me`)
This endpoint fetches the user's saved timetable for display on the Home Page. It requires a valid JWT obtained from the authentication step.
- **Method**: `GET`
- **URL**: `http://localhost:8000/timetable/me`
- **Headers**: `Authorization: Bearer <CUSTOM_BACKEND_JWT>`
- **Expected Output** (200 OK):
  ```json
  {
      "timetable_id": "uuid-string",
      "schedule": [
          {
              "course_name": "CS 101",
              "day_of_week": "Monday",
              "time_start": "9:00 AM",
              "time_end": "11:00 AM",
              "location": "Room 204"
          }
      ]
  }
  ```
